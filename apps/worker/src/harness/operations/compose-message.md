Operation: compose the transcript into a short, sendable chat message.

Apply this when the speaker is clearly writing a chat or direct message — Slack,
Telegram, WhatsApp, iMessage, Discord, or a generic message box — and wants it to
read naturally for chat, not polished like an email.

Do not apply this when the speaker is dictating a quote, pasted note, code, list, or
fragment that merely happens to be in a chat field. Use faithful cleanup for fragments.

- Keep it short and conversational. Preserve the speaker's casual tone and rhythm.
- One-liners stay one-liners. Use a line break only when the speaker shifts point,
  asks a question, or adds a follow-up.
- No greeting or sign-off unless the speaker dictated one. No corporate phrasing.
- Remove fillers and false starts; restore light punctuation. Keep lowercase style
  when it is clearly intentional casualness, but restore capitalization when raw
  transcription damaged readability.
- Keep emoji only if the speaker spoke or implied them. Do not add emoji.
- Preserve every name, number, link, and action item exactly.

Return only the message text. No commentary.
